# Features Summary: v1.6.5 Update

**Date:** January 2, 2026
**Version:** 1.6.5

This update focuses on improving server resilience, observability, and developer experience through automated versioning and a streamlined dashboard.

## 1. Automated Versioning (Single Source of Truth)
- **Mechanism:** The server now automatically imports its version string directly from `package.json` at runtime.
- **Benefit:** Eliminates the need to manually update the version in `src/index.ts` or other files.
- **Usage:**
    - To update the project version, simply run `npm version <patch|minor|major>`.
    - The server startup logs and the web dashboard will strictly reflect this value.

## 2. Condensed Diagnostic Information Dashboard
- **Design:** The root route (`/`) has been completely redesigned to be a "high-performance" system utility dashboard.
- **Key Characteristics:**
    - **Zero Dependencies:** Uses system fonts and standard CSS (no external requests).
    - **Universal Compatibility:** Works on any browser or device.
    - **Automatic Dark Mode:** Respects system color scheme preferences.
- **Features:**
    - **Real-time Status:** Shows "Connected", "Initializing...", or "Disconnected" based on the internal API state.
    - **Performance Metrics:** Displays initialization time (in ms) and connection reuse stats.
    - **Protocol Neutrality:** Renamed "Modern/Legacy" labels to "Streamable Connection" and "Event Stream" to avoid bias.
    - **Security Badge:** Clearly indicates if Bearer Authentication is enabled.

## 3. CI/CD Pipeline (GitHub Actions)
- **New Workflow:** Added `.github/workflows/ci.yml` that triggers on Push and Pull Requests to `main`.
- **Jobs:**
    - **Quality Gate:** Runs `npm run quality` (Biome linting + Type checking) and `npm run build`.
    - **Version Bump Check:** Automatically compares the branch version against `main` and warns if the version hasn't been incremented, prompting better release habits.

## 4. Resilience & Performance Improvements
- **Startup:** The server no longer crashes (`process.exit(1)`) if the Actual Budget API fails to connect immediately. Instead, it stays alive and retries on the next request.
- **Timeouts:** Added a strict 55-second timeout to the initialization process to prevent infinite hanging on Render's free tier.
- **Logging:** Added detailed `[PERF]` logs to pinpoint exactly which step (connection vs. download) is slow during cold starts.
